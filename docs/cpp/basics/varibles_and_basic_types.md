# 变量和基本类型

基本类型和 C 语言基本相同，其内容也较为简单，在此处不作描述。

## 引用

引用是为对象另外起一个名字，引用类型引用另外一种类型。定义引用时，程序把引用和它的初始值绑定，而不是将初始值拷贝给引用。一旦初始化完成，引用将和它的初始值对象一直绑定在一起。因为无法令引用重新绑定到另一个对象，因此引用必须初始化。

```cpp
int ival = 10;
int &rval = ival;
// int &rval2; // 错误，引用类型必须初始化
```

## `const` 限定符

在 C++ 中，`const` 会有一些不同的语言特性。使用 `const` 修饰的对象，一旦创建后其值不能再改变，所有这些对象必须在创建时就初始化。编译器在编译过程中会将用到 `const` 修饰的对象的地方都替换成对应的值，如果程序包含多个文件，则每个用来 `const` 对象的文件都必须得到能访问它的初始值才行。C++ 在默认情况下，`const` 对象被设定为仅在文件内有效，当多个文件同时出现了同名的 `const` 对象，其实等同于在不同的文件中分别定义了独立的变量。如果确实需要将 `const` 对象在文件间进行共享，对于 `const` 变量不管是声明还是定义都要添加 `extern` 关键字，并且只需定义一次。

当 `const` 作用引用或指针类型上时，表示不能通过此引用或指针修改所指对象的值，这并不代表所指对象本身的值不能改变。`const` 还能作用在指针本身上，表示一旦绑定对象以后，就不能更改此指针的指向，这就是常量指针。引用的底层就是一个常量指针，因此引用必须初始化，一旦初始化就不能更改所绑定的对象，这就跟常量指针有关。

```cpp
int ival = 10;
const int &rval = ival;
// rval = 100; // 错误，不能通过此引用修改 ival 的值
ival = 100; // 但是 ival 本身是可以修改的

const int *pval = &ival;
int * const pval2 = &ival;
// pval2 = &rval;  // 错误，一旦绑定了地址之后就不能修改指针的指向
```

对于 `const` 有两个术语：顶层 `const` 和底层 `const`

- 顶层 `const` 可以表示任意的对象是常量，表示对象本身是一个常量
- 底层 `const` 与指针和引用等复合类型的基本类型有关，表示其所指的对象是一个常量

我们还有可能需要值不会发生改变，并且在编译过程就能得到计算结果的表达式，这就是常量表达式。通常字面值，用常量表达式初始化的 `const` 对象都是常量表达式。C++11 新标准为了更好的区分，使用 `constexpr` 关键字修饰变量，以便由编译器来验证变量的值是否是一个常量表达式。

`constepxr` 修饰指针或引用时，对于其初始值有严格的限制，一个 `constexpr` 指针的初始值必须是 `nullptr` 或者 0，或者是存储于某个固定地址中的对象。`constexpr` 定义的指针，只会对指针有效，而与指针所指针的对象无关。

```cpp
const int *p = nullptr;     // p 是一个指向整型常量的指针
constexpr int *q = nullptr; // q 是一个指向整型的常量指针
```

`constexpr` 会将所有的对象置为顶层 `const`。

## 处理类型

### 类型别名

在 C 语言中可以使用 `typedef` 对复杂的类型进行类型别名，这在 C++ 同样适用。但是在 C++11 新标准中提供了一个更加好的方式进行类型别名，使用别名声明的方式。

```cpp
// 1. typedef 的方式
typedef ptrstring char *;

// 2. 别名声明的方式
using ptrstring = char *;
```

如果遇到一条使用了类型别名的声明语句时，不能简单进行类型别名的替换来理解该语句的含义，如 `ptrstring` 表示的是一个指向 `char` 指针类型，所有对 `ptrstring` 的关键字修饰都是作用在指针本身上。

```cpp
const ptrstring cstr = nullptr; // cstr 是一个指向 char 的常量指针
const ptrstring *pstr = &cstr;  // pstr 是一个指针，所指对象是一个指向 char 的常量指针
```

### `auto` 类型说明符

`auto` 在 C 语言和 C++ 中有很大的不同，在 C 语言中表示声明/定义一个自动变量。而在 C++ 中，则是让编译器替我们去分析表达式所属的类型，方便了在实际开发中表示式类型的显示。

```cpp
auto i = 0, *p = &i; // i 被推导出是 int 类型，而 p 被推导出是 int *类型
```

`auto` 并不是简简单单推到类型，编译器推断出来的 `auto` 类型有时候和初始值的类型并不完全一样，编译器会适当地改变结果类型使其更符合初始化规则。

`auto` 在进行类型推导时，是根据表达式的最初类型来推导，比如初始化对象是一个引用类型，引用是起别名，因此其最初类型是某个类型的对象，那么推到出来的类型是和这个对象相同的；如果初始化对象是一个顶层 `const` 类型，并不会该变最初类型的限制，因此 `auto` 会将其忽略，而底层 `const` 则不能忽略。

```cpp
auto a = 10;  // a 是 int 类型
auto b = &a;  // b 是 int * 类型
int &r = a;
auto c = r;   // c 是 int 类型
const int cv = a;
auto d = cv;  // d 是 int 类型
auto e = &cv; // e 是指向常量 int 的指针
```

### `decltype` 类型指示符

如果需要从表达式的类型推断出要定义的变量的类型，但是不想用该表达式的值初始化变量。C++11 新标准引入了 `decltype` 关键字，其作用时选择并返回操作数的数据类型。

```cpp
const int ci = 0, &cj = ci;
decltype(ci) x = 0; // x 的类型是 const int
decltype(cj) y = x; // y 的类型是 const int &
```

如果 `decltype` 使用的表达式不是一个变量，则 `decltype` 返回表达式结果对应的类型。

```cpp
int i = 42, *p = &i, &r = i;
// decltype(r) 的结果是一个引用类型，如果 r 作为表达式的一部分，则将是一个具体值，对应的类型不再是引用
decltype(r+0) b;  // b 的类型是 int
decltype(*p) c;   // c 的类型是 int &, 因此必须初始化
```

!!! warning "`decltype` 和 `auto` 的区别"

    对于 `decltype` 所用的表达式来说，如果变量名加上了一对括号，则得到的类型与不加括号时会有不同。如果 `decltype` 使用的是一个不加括号的变量，则得到的结果就是该变量的类型；如果给变量加上了一层或多层括号，编译器就会把它当成一个表达式，变量是一种可以作为赋值语句左值的特殊表达式，所以这样的 `decltype` 就会得到引用类型。

    ```cpp
    int i = 0;
    decltype(i) a;  // a 的类型是 int
    decltype((i)) b = i;  // b 的类型是 int &
    ```
