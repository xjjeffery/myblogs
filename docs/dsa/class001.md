## 从社会实践到入门提醒

在学些数据结构和算法之前，我们先来做一个有意思的实验，实验内容为：一开始有 100 个人，每个人都有 100 元，在每一轮都做如下的事情 :
 
1. 每个人都必须拿出 1 元钱给除自己以外的其他人，给谁完全随机；
2. 如果某个人在这一轮的钱数为 0，那么他可以不给，但是可以接收；
3. 发生很多很多轮之后，这 100 人的社会财富分布很均匀吗。

!!! info "基尼系数"

    在判断财富是否均匀，可以通过基尼系数：一个社会的基尼系数是一个在 0~1 之间的小数，基尼系数为 0 代表所有人的财富完全一样，基尼系数为 1 代表有 1 个人掌握了全社会的财富。基尼系数越小，代表社会财富分布越均衡；越大则代表财富分布越不均衡。在 2022 年，世界各国的平均基尼系数为 0.44。目前普遍认为，当基尼系数到达 0.5 时，就意味着社会贫富差距非常大，分布非常不均匀，社会可能会因此陷入危机，比如大量的犯罪或者经历社会动荡。

    计算基尼系数的公式：财富差值总和 / (2 * 人数 * 每个人的财富值总和)。

作为一个程序员，当然使用代码完成这个实现，根据此实验的要求，代码实现如下：

```cpp
#include <stdlib.h>
#include <time.h>

#include <iostream>
#include <numeric>
#include <vector>

const int kTestCounts = 1000000;

void experiment(std::vector<int>& values);
double calculateGini(std::vector<int>& values);

int main() {
  srand((unsigned)time(nullptr));
  // 首先为 100 个人每人分配 100 元
  std::vector<int> values(100, 100);
  // 每一轮开始执行任务
  for (int i = 0; i < kTestCounts; ++i) 
    experiment(values);

  for (auto value : values) 
    std::cout << value << " ";

  std::cout << "\ngini = " << calculateGini(values) << std::endl;

  return 0;
}

void experiment(std::vector<int>& values) {
  // 每个人拿出一元给其他人
  for (int i = 0; i < values.size(); ++i) {
    // 如果此人的财富为 0 则不需要给其他人
    if (values[i] <= 0)
      continue;

    // 财富值不为 0，则拿出 1 元给其他人
    int ret = 0;
    do {
      ret = rand() % values.size();
    } while (ret == i);

    values[i]--;
    values[ret]++;
  }
}

double calculateGini(std::vector<int>& values) {
  // 所有财富值总和
  long sum = accumulate(values.begin(), values.end(), 0);
  // 差值总和
  long diff_sum = 0;
  for (int i = 0; i < values.size(); ++i) {
    for (int j = 0; j < values.size(); ++j) {
      diff_sum += abs(values[i] - values[j]);
    }
  }

  return diff_sum / (2.0 * values.size() * sum);
}
```

## 二进制

更对的数字编码和字符编码可以阅读 [hello 算法](https://www.hello-algo.com/)进一步认识理解。

我们最常用的数字编码格式是十进制，但是计算机是不能理解这种编码的，计算机只认识二进制格式的数据（二进制就是只有 $0$ 和 $1$ 的数）。用二进制表示一个有符号类型的数时，我们会发现负数的个数总比正数的个数多一个，例如 `byte` 的取值范围是 $[-128, 127]$，出现这种情况的原因是因为有一个被 $0$ 所占用。其内在原因也是因为二进制的三种表现形式：原码、反码以及补码。

### 原码、反码和补码

!!! info

    计算机中所有的数据都是以补码的形式存储。

- **原码**：对于有符号数，我们将数字的二进制表示的最高位视为符号位，其中 $0$ 表示正数，$1$ 表示负数，其余位表示数字的值。
- **反码**：正数的反码与其原码相同，负数的反码是对其原码除符号位外的所有位进行取反操作。
- **补码**：正数的补码与其原码相同，负数的补码是其在反码的基础上加 $1$ 。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/0b810ec2a5f94697a7bee1a3f0aa63a5.png)

原码虽然最直观，但存在一些局限性，如 **负数的原码不能直接用于运算**。例如在原码下计算 $1 + (-2)$ ，得到的结果是 $-3$ ，这显然是不对的。

$$
\begin{aligned}
& 1 + (-2) \newline
& \rightarrow 0000 \; 0001 + 1000 \; 0010 \newline
& = 1000 \; 0011 \newline
& \rightarrow -3
\end{aligned}
$$

为了解决此问题，计算机引入了反码。如果我们先将原码转换为反码，并在反码下计算 $1 + (-2)$ ，最后将结果从反码转换回原码，则可得到正确结果 $-1$ 。

$$
\begin{aligned}
& 1 + (-2) \newline
& \rightarrow 0000 \; 0001 \; \text{(原码)} + 1000 \; 0010 \; \text{(原码)} \newline
& = 0000 \; 0001 \; \text{(反码)} + 1111  \; 1101 \; \text{(反码)} \newline
& = 1111 \; 1110 \; \text{(反码)} \newline
& = 1000 \; 0001 \; \text{(原码)} \newline
& \rightarrow -1
\end{aligned}
$$

另一方面，**数字零的原码有 $+0$ 和 $-0$ 两种表示方式**。这意味着数字零对应两个不同的二进制编码，这可能会带来歧义。比如在条件判断中，如果没有区分正零和负零，则可能会导致判断结果出错。而如果我们想处理正零和负零歧义，则需要引入额外的判断操作，这可能会降低计算机的运算效率。

$$
\begin{aligned}
+0 & \rightarrow 0000 \; 0000 \newline
-0 & \rightarrow 1000 \; 0000
\end{aligned}
$$

与原码一样，反码也存在正负零歧义问题，因此计算机进一步引入了补码。我们先来观察一下负零的原码、反码、补码的转换过程：

$$
\begin{aligned}
-0 \rightarrow \; & 1000 \; 0000 \; \text{(原码)} \newline
= \; & 1111 \; 1111 \; \text{(反码)} \newline
= 1 \; & 0000 \; 0000 \; \text{(补码)} \newline
\end{aligned}
$$

在负零的反码基础上加 $1$ 会产生进位，但 `byte` 类型的长度只有 8 位，因此溢出到第 9 位的 $1$ 会被舍弃。也就是说，**负零的补码为 $0000 \; 0000$ ，与正零的补码相同**。这意味着在补码表示中只存在一个零，正负零歧义从而得到解决。

还有一个疑惑：`byte` 类型的取值范围是 $[-128, 127]$ ，多出来的一个负数 $-128$ 是如何得到的呢？我们注意到，区间 $[-127, +127]$ 内的所有整数都有对应的原码、反码和补码，并且原码和补码之间可以互相转换。

然而，**补码 $1000 \; 0000$ 是一个例外，它并没有对应的原码**。根据转换方法，我们得到该补码的原码为 $0000 \; 0000$ 。这显然是矛盾的，因为该原码表示数字 $0$ ，它的补码应该是自身。计算机规定这个特殊的补码 $1000 \; 0000$ 代表 $-128$ 。实际上，$(-1) + (-127)$ 在补码下的计算结果就是 $-128$ 。

$$
\begin{aligned}
& (-127) + (-1) \newline
& \rightarrow 1111 \; 1111 \; \text{(原码)} + 1000 \; 0001 \; \text{(原码)} \newline
& = 1000 \; 0000 \; \text{(反码)} + 1111  \; 1110 \; \text{(反码)} \newline
& = 1000 \; 0001 \; \text{(补码)} + 1111  \; 1111 \; \text{(补码)} \newline
& = 1000 \; 0000 \; \text{(补码)} \newline
& \rightarrow -128
\end{aligned}
$$

通过上面的分析发现，所有计算都是加法运算。这暗示着一个重要事实：**计算机内部的硬件电路主要是基于加法运算设计的**。这是因为加法运算相对于其他运算（比如乘法、除法和减法）来说，硬件实现起来更简单，更容易进行并行化处理，运算速度更快。加法运算时，时刻会有溢出出现，所以自己确保自己的调用所得到的结果不会溢出，一定是自己确保的，计算机不会给你做检查。

请注意，这并不意味着计算机只能做加法。**通过将加法与一些基本逻辑运算结合，计算机能够实现各种其他的数学运算**。例如，计算减法 $a - b$ 可以转换为计算加法 $a + (-b)$ ；计算乘法和除法可以转换为计算多次加法或减法。

现在我们可以总结出 **计算机使用补码的原因：基于补码表示，计算机可以用同样的电路和操作来处理正数和负数的加法，不需要设计特殊的硬件电路来处理减法，并且无须特别处理正负零的歧义问题。这大大简化了硬件设计，提高了运算效率**（后续会学到如何使用位运算实现加减乘除运算）。

!!! tip "二进制与十进制转换技巧"

    我们在使用二进制时不需要在意原码、反码以及补码，只需要知道十进制的数如何转换成计算机可以使用的二进制（所谓的补码），又是如何转换回来即可。无符号数以及正数不再赘述（因为都是二进制没有变化），主要说一个十进制负数是如何转换成二进制：首先计算出这个数对应正数的二进制表示，然后将这个二进制减 $1$ ，最后将所有二进制位取反，此时就是计算机中参与运算的二进制（也就是补码）。如何从一个二进制知道是负几：首先高位肯定是 $1$（因为是负数），将所有位进行取反操作，然后将取反后的二进制数加 $1$，此时再将二进制转换成十进制就是该负数所对应的正数，最后加上负号就知道是负几了。

    $$
    \begin{aligned}
    & -125 \; \text{(二进制如何表示)}\newline
    & \rightarrow 0111 \; 1101 \; \text{(125 的二进制)} \rightarrow 0111 \; 1100 \; \text{(进行减 1 操作)} \rightarrow 1000 \; 0011 \; \text{(取反获取计算机存储的形式)} \newline
    \newline
    & 1001 \; 1101 \;  \text{(如何知道这是负几)} \newline
    & \rightarrow 0110 \; 0010 \; \text{(首先进行取反)} \rightarrow 0110 \; 0011 \; \text{(进行加 1 操作)} \rightarrow -99 \; \text{(转换成十进制数后加上负号)}
    \end{aligned}
    $$

## 位运算

### 常见的位运算

常见的位运算有以下几种：

| 位运算符 | 功能 | 例子 |
| --- | --- | --- |
| `\|` | 位或：运算的两个二进制位中只要有一位是 $1$，其结果就是 $1$；如果都是 $0$，结果就是 $0$。所以位或运算一般用来打开位（将某位设置为 $1$） | `1001 \| 1010 = 1011` |
| `&` | 位与：运算的两个二进制位中只要有一位是 $0$，其结果就是 $0$；如果都是 $1$，结果就是 $1$。所以位与运算一般可以用来做掩码（设置某位为 $1$，其余位为 $0$） | `1001 & 1010 = 1000` |
| `^` | 位异或：运算的两个二进制位中只要两个位上的值不同，其结果就是 $1$；如果两个位上的值相同，其结果就是 $0$。所以位异或可以用来改变二进制的位状态 | `1001 ^ 1010 = 0011` |
| `~` | 按位取反：所有二进制位 $0$ 变 $1$，$1$ 变 $0$。所以取反操作一般结合位与进行关闭位（将某一位置为 $0$，其余位不变） | `1010 & ~0010 = 1000` |
| `<<` | 左移运算：二进制位整体向左移动，最低位用 $0$ 来代替 | `1001 << 1 = 0010` |
| `>>` | 右移运算：二进制位整体向右移动，最高位用符号为代替 | `1001 >> 1 = 1101` |

!!! info "知识扩展"

    在有些编程语言中还会有无符号右移（`>>>`），如 Java，其在右移过程中最高位是用 $0$ 来代替。

### 相反数

如果我们需要获取一个数的相反数，十进制表示方式是直接在该数前加一个负号，那么这个操作在计算机内部是如何实现的 —— 计算机内部是将该数的二进制数先进行整体取反操作，然后在加上一个 $1$，此时的结果就是相反数的结果（与上面的技巧中有点类似）。这种相反数的获取方式在一些算法中是非常有用的技巧，后续会有遇到。

但是我们会发现一个现象，除了整数最小值以外的所有数都有相对应的正数和负数，而整数最小值是没有对应的整数，那么它的相反数是几 —— 我们以八位整数为例：

$$
\begin{aligned}
& (-128) \rightarrow 1000 \; 0000 \; \text{(原码)} \newline
& \rightarrow 0111 \; 1111 \; \text{(取反)} \newline
& \rightarrow 1000 \; 0000 \; \text{(+1)} \newline
& \rightarrow -128
\end{aligned}
$$

通过上述的计算流程发现，整数最小数的相反数还是自己，这是它的特殊性。

### 获取一个数的所有二进制位

如果要想获取一个数的所有二进制位，可以通过位与运算对每一个位进行判断，根据判断结果进行输出，代码示例如下所示：

```cpp
void printBinary(int num) {
  for (int i = 31; i >= 0; --i) {
    std::cout << (num & (1 << i)) ? "1" : "0";
  }
  std::cout << std::endl;
}
```
